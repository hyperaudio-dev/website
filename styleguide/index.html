Meet the Styleguide.

Welcome to the Hyperaudio Styleguide (later called HS). The purpouse of this styleguide is to document all, even tiniest, elements of Hyperaudio’s user interface. HS is divided in clearly defined chapters listed on the left hand side.

HS requires Hyperaudio base stylesheet, hence all that works within the HS will work on the site or any of the apps as well (and vice-versa).

Modules documented in the HS (especially components and combos) are build with flexibility in mind, hence they will adapt to the bounding box. For that it is important to keep the layout classes and presentational classes separated. More on that in the Grid section.

Dependecies.

Hyperaudio UI architecture depends principally on Compass for stylesheet automation and compilation. Bower instead is used to manage vendor frameworks (Foundation), libraries (jQuery) and other plugins (jquery.cycle2.js).

Running the thing.

$ gem install compass
$ npm install -g bower
$ cd project_dir
$ bower install
$ compass watch

Stylesheet architecture.

Whenever any style changes are required, enter project directory, follow instructions found above (Running the thing) and make necessary edits in the /assets/stylesheets/sass directory. CSS files will automagically be compiled and minimized.

Site vs. Apps.

Site’s and apps’ body use context classes for any differentiations when needed:

body.tpl_site
body.tpl_app

Keep the Styleguide clean, sweet and up-to-date.
